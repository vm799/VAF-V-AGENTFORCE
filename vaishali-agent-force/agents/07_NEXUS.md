## Core Technologies NEXUS Masters

### A2A (Agent-to-Agent Protocol) — Google
**What:** An open protocol allowing AI agents from different vendors to communicate, delegate tasks, and negotiate on behalf of users.
**Why it matters for V:** A2A enables an internet where your AI agent books your flights, negotiates your bills, and manages your finances by talking to other agents. The tooling layer around this is EMPTY.

**V should build:**
- A2A-compatible agent framework for financial services
- A2A testing/debugging tools (open-source → paid enterprise)
- A2A compliance/audit layer for regulated industries (banks, insurance)

### MCP (Model Context Protocol) — Anthropic
**What:** A protocol that lets AI models connect to external tools, data sources, and services through a standardised interface.
**Why it matters for V:** Every AI application needs MCP servers. The ecosystem is exploding but immature. Building high-quality MCP servers = instant visibility.

**V should build:**
- MCP servers for financial data (Bloomberg, Reuters, FCA data)
- MCP servers for UK banking APIs (Open Banking, Monzo, Starling)
- MCP server marketplace / quality rating system
- Teaching content: "How to build your first MCP server in 30 minutes"

### Agent Payments & Financial Infrastructure
**What's coming:** Agents will need to make payments, hold balances, verify identity, and comply with financial regulations — autonomously.
**Why V is uniquely positioned:** She works in asset management AND builds AI. She understands both sides.

**V should build:**
- Agent wallet / escrow system for A2A transactions
- KYA (Know Your Agent) — agent identity verification for financial compliance
- Agentic payment rails — Stripe for agents (API-first, programmatic)
- Financial compliance layer for autonomous agents (FCA/PSD2/Open Banking aligned)
