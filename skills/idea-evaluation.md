# Idea Evaluation — Rapid Revenue Stream Assessment

> Loaded by ATLAS when V drops a business idea, side hustle, or passive income opportunity.
> Goal: Go from "I have an idea" to a BUILD / PARK / SKIP verdict in under 5 minutes.

---

## When This Skill Activates

- V says: "I want to build X for money", "Is Y worth it?", "What's the ROI on Z?"
- V drops a business idea, revenue opportunity, or side hustle concept
- V asks about passive income, new ventures, or monetisation of a skill/asset
- V asks "where do I start?" on something that sounds like a business

---

## Step 1: Clarify the Idea (30 seconds)

Before evaluating, confirm you understand:
- **What**: What is V proposing to build/do?
- **Who**: Who pays? (Advertisers, customers, subscribers, sponsors?)
- **How**: What's the revenue model? (Ad revenue, subscriptions, sales, licensing?)
- **Automation**: Can AI do 80%+ of the ongoing work?

If any of these are unclear from V's message, ask ONE clarifying question — not four.

---

## Step 2: Run the 7-Point Assessment

```
ATLAS VENTURE EVAL — [Idea Name]
============================================================

1. MARKET
   Size: [TAM estimate]
   Competition: [Saturated / Competitive / Open / Blue ocean]
   CPM/Unit economics: [Revenue per unit of output]

2. AUTOMATION FEASIBILITY
   Can AI do 80%+? [Yes/Partial/No]
   Key tools: [Specific APIs and services needed]
   Human-in-the-loop: [What still needs V's attention?]

3. TIME TO FIRST REVENUE
   First £1: [X weeks/months]
   First £1k/month: [X months]
   Blockers to revenue: [Platform requirements, audience size, etc.]

4. COST vs RETURN
   Monthly operating cost: £[X]
   Projected month 6 revenue: £[X]
   Projected month 12 revenue: £[X]
   Break-even: Month [X]

5. COMPOUNDING EFFECT
   Does output accumulate? [Yes/No — explain]
   Evergreen or time-sensitive? [Evergreen = keeps earning / Time-sensitive = decays]
   Network effects? [More content = more discovery = more views?]

6. MOAT FIT
   Uses finance expertise? [Yes/No]
   Uses AI/building skills? [Yes/No]
   Creates teaching content? [Yes/No]
   Unique angle V has: [What makes V's version different?]

7. OPPORTUNITY COST
   Hours/week required: [X]
   What V gives up: [Time that could go to consulting/teaching/enterprise pipeline]
   Can it run alongside current priorities? [Yes/No]

============================================================
VERDICT: [BUILD / PARK / SKIP]
CONFIDENCE: [High / Medium / Low]
WHY: [One sentence — the core reason]
RISK: [Biggest single risk]
FIRST 48-HOUR ACTION: [One specific, concrete step]
============================================================
```


---

## Verdict Definitions

| Verdict | Meaning | Criteria |
|---------|---------|----------|
| **BUILD** | Start now. Positive ROI within 6 months. | Automatable + compounds + manageable cost + fits bandwidth |
| **PARK** | Good idea, wrong time. Revisit when [condition]. | Interesting but V lacks bandwidth, a prerequisite, or the timing is off |
| **SKIP** | Don't pursue. Opportunity cost too high. | Can't automate + doesn't compound + ROI negative or marginal |

## Multi-Agent Consultation Tags

If the idea needs deeper input, ATLAS tags:
- `@PHOENIX` — financial projections, tax implications, cost modelling
- `@NEXUS` — automation feasibility, API landscape, tech stack recommendations
- `@AMPLIFY` — content/audience strategy, platform selection, growth tactics
- `@FORGE` — build scope, timeline, architecture, repo structure
- `@SENTINEL` — priority fit against current goals and available bandwidth

ATLAS delivers the verdict. Other agents provide supporting analysis only when tagged.

## Examples

**"I want to start a kids YouTube channel with AI-generated videos"**
→ ATLAS runs 7-point eval → BUILD (high automation, evergreen, compounds, $120-400/mo cost, $3-15k/mo by month 12)
→ Tags @NEXUS for API stack, @AMPLIFY for channel strategy, @FORGE for pipeline build

**"Should I start dropshipping?"**
→ ATLAS runs 7-point eval → SKIP (low automation, high competition, thin margins, doesn't use V's moat)

**"What about building an MCP server marketplace?"**
→ ATLAS runs 7-point eval → PARK (great idea, but V needs 2-3 published MCP servers first for credibility)
→ Tags @NEXUS for ecosystem analysis
