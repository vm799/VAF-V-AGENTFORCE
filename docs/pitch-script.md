### What I Built — Personal OS (1:30–3:00)

> "VAF has two layers. The first is what I call Claude Code Contexts — ten specialist agents, each with a code name and a domain. When I talk about money, PHOENIX activates and responds like a personal CFO. When I'm overwhelmed and don't know where to start, SENTINEL takes over and orchestrates everything. FORGE handles architecture. AMPLIFY handles content. CIPHER is my learning intelligence — it processes everything I read and extracts what actually matters.
>
> These aren't ten separate apps. They're ten behavioural contexts — markdown files that define how Claude operates in each domain. One codebase. One model. Ten perspectives.
>
> The memory layer is what makes it real. Through an MCP server connected to my Obsidian vault, every session ends by crystallising the key decisions, insights, and context. Next session, Claude loads them back. It's not perfect recall — it's structured recall. The difference is everything.
>
> I wake up, open my dashboard, run a morning briefing. In 90 seconds I know: where my goals stand, what my money is doing, what I need to ship today. My Telegram bot captures everything I drop throughout the day. No friction. No context switching. Just a system that works."

---

### What I Built — Enterprise Pipeline (3:00–4:15)

> "The second layer is the enterprise pipeline. Same architecture philosophy, completely different scale.
>
> Nine Python modules. Each does one thing precisely. Data comes in — raw, messy, from multiple sources. Build 01 ingests it. Build 02 sanitises it. Build 03 normalises it. Build 04 enriches it with context and relationships. Build 05 analyses it for patterns. Build 06 synthesises those patterns into actionable insights.
>
> Then Build 06 — the one I'm most proud of — runs a Council. Four AI agents: a Bull who argues the opportunity, a Bear who argues the risk, a Risk analyst who stress-tests both positions, and a Synthesiser who writes the brief. They don't just agree — they're designed to disagree. The tension produces better analysis than any single model.
>
> Build 07 ranks insights by impact. Build 08 formats them for your stakeholders — C-suite gets the two-minute brief, analysts get the full report. Build 09 delivers across email, Slack, Teams, or dashboard. One command. Full pipeline. Done.
>
> I run this with one command: `./demo.sh`. I'll show you that in a second."
